# 🌍 AI Travel Planner — LLM API Demo

**Generate smart, detailed travel itineraries using modern Large Language Model APIs.**

---

## 📋 What is This?
A simple but professional portfolio project that demonstrates connecting Python to real LLM APIs (Google Gemini, OpenAI, Anthropic, or OpenRouter). You enter a destination, budget, days, and interests, and get a daily plan with activities, local tips, and cost breakdown — all generated by AI.

- Supports **multiple LLM providers** — see how cleanly switching APIs is handled.
- Designed to look well-structured and show practical API use, not just hacks.
- Includes a web app (Gradio) plus testable Python code.

---

## 🌟 Features
- 🗺️ **Smart AI Itinerary:** Enter “3 days in Paris, $700, food & art” and get back a well-formatted schedule, costs, plus local advice.
- 🔁 **Multi-Provider LLM:** Code lets you use Gemini, OpenAI, Anthropic, or OpenRouter APIs by just changing a config/key (see `.env.example`).
- 🚦 **Error Handling:** If your primary provider fails, it can try others (no crash, no ugly logs).
- 🤖 **Web Interface:** Use the planner via browser, or directly in Python code.

## 🚀 Try It Yourself

### 1. Clone and Install
```bash
git clone https://github.com/KamdemSamuel/ai-travel-planner-gemini.git
cd ai-travel-planner-gemini
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Set Up API Keys
- Copy `.env.example` to `.env`
- Put at least one valid key (Google Gemini, OpenAI, etc)

### 3. Start the Web App
```bash
python main.py
```
Go to `http://localhost:7860` in your browser.

## 🧑‍💻 Use in Python
```python
from travel_planner import TravelPlanner
planner = TravelPlanner()
plan = planner.generate_itinerary(
    destination="Istanbul",
    budget="$600",
    days=3,
    interests="food, mosques, history"
)
print(plan)
```

## 🛠️ How it Works
- **LLM Vendor Switch:** Just change provider in `.env` or code (`provider=LLMProvider.OPENAI` or `provider=LLMProvider.GOOGLE` etc).
- **Modular Design:** Separation between config, API, logic, web UI (`main.py`), and prompt templates.
- **No Real API Keys in Repo:** Safe to share your code.
- **Basic Tests and Examples:** See `*.ipynb` and sample scripts.
